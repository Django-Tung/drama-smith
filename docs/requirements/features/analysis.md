# 结构化分析(FR-A)—— 本期核心

> 所属:drama-smith 需求 · 功能点细化 · v0.2 · 2026-06-26
> 父文档:[system-requirements.md](../system-requirements.md)
> 关联编号:FR-A · 子系统:analysis(本期唯一落地子系统)

## 1. 概述

输入剧本,产出**角色、情节线、冲突、节奏**的结构化拆解,并辅以**图像/视频增强**。文本拆解同时切出**分镜(3-15 秒)**,分镜既是结构化输出的一部分,也是视频生成的输入单元。

## 2. 需求条目

| 编号 | 需求 | 说明 | 模型 |
|------|------|------|------|
| FR-A1 | 剧本输入 | 支持粘贴文本 + 上传常见剧本/文本格式(格式清单见 §7) | — |
| FR-A2 | 文本结构化拆解 | 输出角色/情节线/冲突/节奏四维;并**按剧情拆分分镜(每镜 3-15 秒)**(输出模型见 §3;切分规则见 §5) | 文本 |
| FR-A3 | 图像增强 | 为关键角色生成立绘/形象卡,为关键冲突/高潮场次生成场景概念图(**需用户已配置图片模型,否则该功能不可用**) | 图片 |
| FR-A4 | 分镜动态预览 | 依据 FR-A2 拆出的分镜,为分镜生成 **3-15 秒动态镜头预览**(**需用户已配置视频模型,否则该功能不可用**) | 视频 |
| FR-A5 | 结果查看与导出 | 分析结果可在前端查看,并支持导出(格式见 §7) | — |

> FR-A3/A4 的"为哪些角色/场次生成、生成什么"是建议形态,需与提示工程、成本策略一同校准(见 §7)。**FR-A3/A4 仅在用户已配置相应(图片/视频)模型时可用,否则入口禁用并引导补配**(见 [`ai-config.md`](./ai-config.md) FR-C1)。三类模型的配置与自检见 [`ai-config.md`](./ai-config.md)。

## 3. 输出模型(细化草案)

- **角色(Character)**:姓名/代号 · 身份定位(主角/配角/反派…) · 一句话人设 · 动机与目标 · 性格特质(标签) · 关键场次 · 与他人关系。
- **情节线(Plot Line)**:线索名称(主线/副线 A/B…) · 类型(情感/事业/悬疑…) · 涉及场次范围 · 涉及角色 · 起止与走向。
- **冲突(Conflict)**:类型(人际/内心/环境/利益…) · 对立方(A vs B / A vs 自我) · 触发场次 · 烈度 · 解决方式与节点。
- **节奏(Pacing)**:整体曲线与幕结构 · 逐幕节奏标签(快/慢/紧/缓) · 高潮分布 · 拖沓/失衡点诊断。
- **分镜(Shot)**:按剧情顺序的镜头清单,每镜 3-15 秒;含描述/景别/对应场次/剧情点与情绪/出场角色;与情节线、冲突可追溯——亦为 FR-A4 的输入单元。
- **富媒体产物**:角色立绘、关键场面概念图(A3);逐分镜 3-15 秒动态预览(A4)。与角色/场次/分镜**可追溯地关联**。

## 4. 页面与旅程

- **分析工作台**:粘贴/上传剧本,发起分析。
- **分析结果页**:四维拆解 + 分镜列表 + 图/视频产物 + 导出。
- **我的历史**:该用户的剧本与分析记录(用户隔离)。
- **旅程**:登录 → 工作台提交剧本 → 流式查看进度(文本 → 分镜 → 图 → 视频)→ 结果页浏览 → 导出。

## 5. 分镜切分规则(业务约束)

- **切分单元**:以**剧情节拍 / 场面切换 / 对白转折**为切分点,按剧情顺序产出镜头清单;分镜顺序 = 剧情推进顺序。
- **时长约束**:每镜目标 **3-15 秒**(由文本模型按对白与动作估算)。
  - 估算 **< 3 秒**:与相邻镜头合并,或标注"过短"待人工确认。
  - 估算 **> 15 秒**:按节拍内部再拆分为多镜。
- **每镜必含**:序号 · 镜头描述 · 景别(全/中/近/特写) · 对应场次 · 剧情点与情绪 · 出场角色 · 目标时长(可选:运镜)。
- **可追溯**:每镜关联情节线、冲突(若有),供 FR-A4 取用、结果页可定位。
- **节奏量化**:分镜的时长分布与密度,作为节奏(Pacing)曲线的量化输入。

## 6. 数据流

1. 前端提交剧本 → 后端创建**分析任务**(归属当前用户),持久化。
2. **文本拆解(FR-A2)**:文本模型产出 角色/情节线/冲突/节奏 + 切分分镜。
3. **图像(FR-A3)**:图片模型为指定角色/关键场面生成立绘/概念图。
4. **视频(FR-A4)**:视频模型按分镜生成 3-15s 动态预览。
5. 产物**持久化(用户隔离)**,结果页聚合渲染;长步骤**流式回传进度**(通道见 [`architecture`](../../architecture/system-architecture.md) §4.3)。
6. **导出**:按选定格式打包下载。

> 步骤 2→3→4 可串行或部分并行;编排、失败重试、超时与配额控制随变更定(编排见 architecture)。
