# 结构化分析与分镜视频生产流水线(FR-A)—— 本期核心

> 所属:drama-smith 需求 · 功能点细化 · v0.4 · 2026-06-30
> 父文档:[system-requirements.md](../system-requirements.md)
> 关联编号:FR-A · 子系统:analysis(本期核心 · 唯一落地子系统)
> 变更:v0.2 仅「结构化分析 + 富媒体增强」→ **v0.3 扩写为完整生产流水线:剧名/剧集 → 剧本输入(AI 优化)→ 角色 → 文本拆解 → 分镜 → 分镜编辑(图)→ 逐镜视频 → 合并成片**;**音频(配音/BGM/音效)本期不纳入**。**v0.4 新增任务中心 / 任务页(FR-A11):长任务统一持久化、任务页跨剧集汇总,状态经 REST 轮询(基线)+ WebSocket 实时回传(§4.5 / §5.4 / §7 / §8)**。

## 1. 概述

围绕**已有剧本**,打通一条「**结构化拆解 → 分镜 → 视觉素材 → 逐镜视频 → 合并成片**」的生产流水线:

- 以**剧(Drama)→ 剧集(Episode)**两级为组织容器:用户先建剧、再在剧下建剧集,每个剧集独立走完整流水线。
- **剧本输入**(粘贴/上传),可选 **AI 优化**;**剧本角色**可由用户预先添加,并与结构化拆解产出的角色融合。
- **文本结构化拆解**产出 角色/情节线/冲突/节奏,并**按剧情拆分分镜(每镜 3-15 秒)**。
- 分镜可**交互式编辑**;每镜/每角色的**视觉素材(图)**可上传或生成**多张候选**供选用,**跨镜角色形象一致**。
- 依据分镜(及素材)逐镜生成 **3-15 秒视频(多候选)**,再**按顺序合并为完整剧集成片(本期无声)**。

> **本期不做音频 / 字幕**:不产生对白配音(TTS)、背景音乐、音效;分镜「对白」以**文本**形式保留于分镜台本,成片为**无声视频**、**不叠加字幕**。音频与字幕均留待后续。

> 各阶段所需模型(文本/图片/视频)由用户自带(BYOK);**图片/视频类阶段仅在用户已配置相应模型时可用,否则入口禁用并引导补配**(见 [`ai-config.md`](./ai-config.md) FR-C1)。三类模型配置与自检见 [`ai-config.md`](./ai-config.md)。

| FR-A4 | 剧本角色管理 | 用户可预先**添加/编辑角色**(姓名、身份、人设、形象描述、关系),并与 FR-A5 拆解产出的角色**融合**(去重/合并/人工校对);亦可**从公共角色库引入**或**加入角色库**(见 [character-library.md](./character-library.md)) | — |

## 3. 输出模型(细化草案)

- **角色(Character)**:姓名/代号 · 身份定位(主角/配角/反派…) · 一句话人设 · 动机与目标 · 性格特质(标签) · 关键场次 · 与他人关系 · **形象描述**(供 FR-A7 一致性生成/参考)。角色可源自**用户预置(FR-A4)**、**拆解产出(FR-A5)**或**公共角色库**。
- **情节线 / 冲突 / 节奏**:线索名/类型/涉及场次/走向;冲突类型/对立方/烈度/解决;节奏曲线/幕结构/高潮分布/失衡诊断。
- **分镜(Shot)**:按剧情顺序的镜头清单,每镜 3-15 秒;含 序号 · 镜头描述 · 景别(全/中/近/特写)· 对应场次 · 剧情点与情绪 · 出场角色 · **对白(文本)** · 目标时长 · (可选)运镜;与情节线/冲突可追溯——为 FR-A7/A8/A9 的输入单元。
- **视觉素材(Image)**:角色形象图、单镜场景/构图图;**每镜可多张候选**;与角色/分镜**可追溯关联**,同一角色跨镜复用同一形象参考。
- **视频(Video)**:逐镜 3-15s 视频,**每镜可多个候选**;**成片(Final)**:各镜用户选定的视频按序合并,本期无声、不叠加字幕。

## 4. 关键能力细化

### 4.1 剧本 AI 优化(FR-A3)

- **触发**:用户主动发起,可选步骤,不强制。
- **优化范围**:**格式规范化、错别字/标点、对白润色、节奏/结构建议**(具体提示与校准随变更定)。
- **用户控制**:展示**优化前后比对**,可**接受/拒绝/部分采纳**;优化结果保留为剧本**新版本**(见 §6)。

### 4.2 角色管理与一致性(FR-A4 / FR-A7)

- 角色来源:用户预置(FR-A4)、拆解产出(FR-A5),或**从公共角色库引入**(见 [character-library.md](./character-library.md));预置角色与拆解产出角色**融合**:名称/身份相近时**提示合并**,由用户确认。
- **角色形象一致性**:角色在**剧集(剧本)层级**绑定——每个角色持有一份「**形象参考**」(上传或生成),固定后在该集**所有分镜素材与视频生成**中复用,避免跨镜「变脸」。约束的具体实现(参考图 / IP-adapter / 提示词固化)属架构决策(见 architecture)。

### 4.3 分镜编辑(FR-A6)

- 可编辑字段:镜头描述、景别、目标时长、出场角色、对白、运镜。
- 可操作:新增、删除、上/下排序、**拆分一镜为多镜**、**合并相邻镜**;编辑后重新校验 3-15s 时长约束(见 §5.1)。

### 4.4 画幅与风格(整集统一)

- **创建剧集(剧本)时**由用户设定**画幅**与**画面风格基调**,作为整集统一约束,贯穿 FR-A7 素材与 FR-A8 视频生成(约束实现见 architecture)。

### 4.5 任务中心与进度回传(FR-A11)

流水线中所有**耗时不确定、需异步等待**的步骤统一抽象为**任务(Task)**:剧本 AI 优化(FR-A3)、文本拆解(FR-A5)、图片生成(FR-A7)、逐镜视频生成(FR-A8)、合并成片(FR-A9)。任务**持久化为记录**(归属当前用户、用户隔离),用户可关闭页面 / 断线后**回来继续查看**,已落地产物不丢。

- **任务生命周期**:`pending`(排队)→ `running`(执行中,带**进度百分比**与**当前阶段**)→ `succeeded` / `failed`(含错误信息)/ `canceled`;记录起止时间、所属剧集与阶段、触发类型(单步 / 批量)、关联产物。
- **任务页(跨剧集聚合)**:独立入口,列出当前用户**近期任务**,可按状态 / 剧集 / 时间过滤;每条展示进度与错误,并可**跳回对应剧集工作台的产物位置**。剧集工作台内各步骤进度仍在原地展示,任务页是其**全局汇总视图**。
- **状态回传(双通道,见 [`architecture`](../../architecture/system-architecture.md) §4.3/§6)**:
  - **REST 轮询(基线)**:`GET /api/tasks`、`GET /api/tasks/:id` —— 始终可用、断线友好;任务页在后台或 WebSocket 不可达时**降级为轮询**。
  - **WebSocket 实时(增强)**:任务页前台打开时**订阅** `/ws/tasks` 进度推送,免去高频轮询;断线自动回退轮询。
  - 两条通道**共享同一任务记录**,前端择一即可、**不依赖连接存活**——这是「可关页面回来看」的保证。
- **取消 / 重试 / 重做**:`running` 任务可**取消**(已落地产物保留);`failed`/`canceled` 可**单条重试**(复用原输入);**单步可独立重做**(单图 / 单镜 / 单视频),不必整条重跑(衔接 §4.3 / §5.2)。

> 任务中心为**流水线级基础设施**;未来剧本生成(FR-G1)、多角色模拟(FR-S1)的长任务可复用同一任务记录与任务页。

## 5. 业务规则

### 5.1 分镜切分规则

- **切分单元**:以**剧情节拍 / 场面切换 / 对白转折**为切分点,按剧情顺序产出镜头清单;分镜顺序 = 剧情推进顺序。
- **时长约束**:每镜目标 **3-15 秒**(由文本模型按对白与动作估算)。
  - 估算 **< 3 秒**:与相邻镜合并,或标注「过短」待人工确认。
  - 估算 **> 15 秒**:按节拍内部再拆分为多镜。
- **每镜必含**:序号 · 镜头描述 · 景别(全/中/近/特写)· 对应场次 · 剧情点与情绪 · 出场角色 · 对白(文本)· 目标时长(可选:运镜)。
- **可追溯**:每镜关联情节线、冲突(若有),供 FR-A7/A8/A9 取用、结果页可定位。
- **节奏量化**:分镜时长分布与密度,作为节奏(Pacing)曲线的量化输入。

### 5.2 角色形象一致性(剧集层级)

- 角色在**剧集(剧本)层级**绑定唯一「形象参考」;FR-A7 素材与 FR-A8 视频生成**引用该参考**保持跨镜一致。
- 替换/重新生成某角色的形象参考后,相关素材/视频可**一键重生成**。

### 5.3 视频合并规则

- 合并顺序 = 分镜顺序(剧情推进顺序);**各镜取用户在 FR-A7/A8 选定的那条**。
- 镜间衔接:**本期仅硬切**(淡入淡出等转场后续支持)。
- 全片画幅、画面风格、镜头语言基调需统一(创建剧集时设定,见 §4.4)。

### 5.4 任务并发、取消与重试(FR-A11)

- **并发**:同用户可并发多个任务(单步 / 批量);批量生成前给**成本预估与确认**(BYOK,见 §6)。后端编排、并发上限、排队策略随变更定(见 architecture)。
- **取消**:`running` 任务可由用户取消;取消时**已落地产物(已完成的图 / 视频)保留**,未完成者终止,任务置为 `canceled`。
- **重试**:`failed`/`canceled` 任务可单条重试,复用原输入;**单步可独立重做**(单图 / 单镜 / 单视频),不必整条重跑(衔接 §4.3 / §5.2)。

## 6. 数据流 / 状态

1. 用户建剧 → 建剧集(FR-A1),并设定画幅与风格基调(§4.4)。
2. 剧集内输入剧本(FR-A2),可选 AI 优化(FR-A3)、预置/引入角色(FR-A4)。
3. **文本拆解(FR-A5)**:产出 角色/情节线/冲突/节奏 + 切分分镜;与预置角色融合。
4. 用户**编辑分镜(FR-A6)**。
5. **视觉素材(FR-A7)**:逐镜/逐角色 上传 或 图片模型生成(多张候选,选用)。
6. **逐镜视频(FR-A8)**:视频模型按分镜(及素材)生成 3-15s(多个候选,选用)。
7. **合并成片(FR-A9)**:按顺序合并各镜选定视频为无声剧集视频(不叠字幕)。
8. 长步骤统一抽象为**任务**(FR-A11)、**持久化(用户隔离)**并落产物;状态经 **REST 轮询 + WebSocket 实时**回传(见 §4.5 与 [`architecture`](../../architecture/system-architecture.md) §4.3/§6);任务页跨剧集汇总(§7)。
9. **导出(FR-A10)**:成片 + 台本 + 角色卡 等打包下载。

> 各步骤可串行或部分并行;编排、失败重试、超时随变更定(见 architecture)。**单步可独立重做**:单张图/单镜/单个视频重生成,不必整条重跑(见 §4.3 / §5.2)。FR-A7/A8 不设硬性数量上限、支持多候选;**批量生成前给用户成本预估与确认**(BYOK)。

> **版本与历史**:剧本(FR-A2/A3)、分镜(FR-A6)、生成产物(图/视频)保留版本,可回退/比对。

## 7. 页面与旅程

- **我的剧库**:剧列表 → 进剧看剧集列表;新建剧/剧集(建剧集时设画幅与风格)。
- **剧集工作台**:剧本输入(AI 优化、角色预置)→ 发起拆解 → 流式进度(拆解 → 分镜 → 素材 → 逐镜视频 → 合并)。
- **任务中心(任务页)**:跨剧集汇总当前用户的近期任务(优化 / 拆解 / 图片 / 视频 / 合并)及状态、进度、错误;双通道(轮询 + WebSocket)刷新;点击跳回对应剧集产物位置(FR-A11)。
- **分镜编辑台**:分镜清单,逐镜编辑、多候选素材上传/生成与选用、多候选逐镜视频生成/重做与选用。
- **成片页**:合并成片预览 + 各阶段产物 + 导出。
- **旅程**:登录 → 建剧/建剧集(设画幅风格)→ 输入剧本(可选 AI 优化 / 预置角色)→ 拆解 → 编辑分镜 → 生成素材 → 逐镜视频 → 合并成片 → 导出(各耗时步骤可去**任务页**看进度 / 失败重试)。

## 8. 接口(草案)与格式

**端点**(字段契约随变更定;均要求携带 token;均归属当前用户):

| 资源 | 端点示例 | 说明 |
|------|----------|------|
| 剧 / 剧集 | `POST/GET /api/dramas` · `POST/GET /api/dramas/:dramaId/episodes` · `GET/PUT/DELETE /api/episodes/:id` | 剧与剧集增查改、剧集排序;建剧集含画幅/风格 |
| 剧本 | `PUT /api/episodes/:id/script` · `POST /api/episodes/:id/script/optimize` | 写入剧本 · AI 优化(返回比对/版本) |
| 角色 | `GET/POST/PUT/DELETE /api/episodes/:id/characters` | 剧集角色 CRUD + 形象参考 |
| 分析任务 | `POST /api/episodes/:id/analyze` · `GET /api/episodes/:id/analysis` | 发起拆解 · 取结果 |
| 分镜 | `GET /api/episodes/:id/shots` · `PATCH /api/shots/:id` · `POST /api/shots/:id/split` · `POST /api/shots/:id/merge` | 分镜清单 · 编辑 · 拆 / 合 |
| 视觉素材 | `POST /api/shots/:id/media` · `POST /api/characters/:id/image` | 单镜素材(多候选)· 角色形象(上传或生成) |
| 视频 | `POST /api/shots/:id/video` | 逐镜视频生成 / 重生成(多候选) |
| 合并 | `POST /api/episodes/:id/render` | 合并成片(取各镜选定视频) |
| 导出 | `POST /api/episodes/:id/export` | 按格式导出 |
| 流式进度 | `GET /api/episodes/:id/stream` · WebSocket `/ws/tasks` | REST 流式进度(兼容回退)· **WS 实时推送任务进度**(见 §4.5) |
| 任务中心 | `GET /api/tasks` · `GET /api/tasks/:id` · `POST /api/tasks/:id/cancel` · `POST /api/tasks/:id/retry` | 跨剧集任务列表 / 详情(**轮询基线**)· 取消 · 重试 |

**格式(首发)**

- **剧本输入(FR-A2)**:纯文本 + Markdown(+Fountain);`.docx` / `.pdf` 推迟。
- **导出(FR-A10)**:成片 mp4 + 分镜台本 Markdown / JSON(+角色卡);PDF 推迟。

> 长任务状态回传采用 **REST 轮询(基线)+ WebSocket 实时推送**(§4.5);编排、并发上限、重试 / 取消属技术决策,见 [`architecture`](../../architecture/system-architecture.md) §4.3/§6。

> **范围边界**:本流水线是「**对已有剧本的结构化拆解与视频生产**」,**不等于**「剧本生成(generation,从主题/大纲生成剧本,FR-G1,本期推迟)」。
