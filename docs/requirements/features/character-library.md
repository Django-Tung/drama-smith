# 公共角色库(FR-L)

> 所属:drama-smith 需求 · 功能点细化 · v0.2 · 2026-06-26
> 父文档:[system-requirements.md](../system-requirements.md)
> 关联编号:FR-L · 切面:复用资产(角色)
> 变更:v0.1 新增(本期纳入)· **v0.2 裁定 6 项待澄清:不做跨用户分享 / 复制不追溯 / 不去重 / 图片 ≤ 1MB 且压缩 / 年龄枚举 / 不设容量上限**。与 [analysis.md](./analysis.md) 的「剧集角色(FR-A4)」**双向互通**:库 ↔ 剧集。

## 1. 概述

一个**归属当前用户、跨剧复用**的角色库:用户可在库中**直接创建角色**,也可**把某剧集里的角色「加入库」**沉淀为可复用资产。库角色独立于任何剧/剧集存在,可在创建/编辑剧集角色时**引入**为该剧的剧本角色([analysis.md](./analysis.md) FR-A4),避免跨剧重复录入同一人设。

> 「公共」指**同一用户跨自身剧目的公共池**(受 NFR-7 多租户隔离约束),**本期不做跨用户分享**(已裁定,见 §8)。

库角色是一份**轻量角色档案**:

- **图片**:图片模型生成 或 手动上传(二选一,可替换)。
- **姓名**、**年龄**、**描述**。
- **是否主角**(主角 / 配角标记)。

> 图片**生成**路径需用户已配置图片模型(门禁见 [ai-config.md](./ai-config.md) FR-C1);**上传**不受此限。库角色的图片可作为该角色**引入剧集后的形象参考**,供 FR-A7 跨镜一致性生成复用(见 [analysis.md](./analysis.md) §5.2)。

## 2. 需求条目

| 编号 | 需求 | 说明 |
|------|------|------|
| FR-L1 | 库角色 CRUD | 在库中创建/查看/编辑/删除角色(字段见 §3);库归属当前用户、用户隔离(NFR-7) |
| FR-L2 | 直接创建角色 | 用户在库中手动新建角色:图片(生成 / 上传)+ 姓名 + 年龄 + 描述 + 是否主角 |
| FR-L3 | 从剧本角色加入库 | 把某剧集([analysis.md](./analysis.md) FR-A4)下的角色**「加入角色库」**:按库档案字段抽取/补全后**复制**存入;原剧集角色不受影响(复制不引用,见 §5) |
| FR-L4 | 角色图片 | 每个角色绑定一张图片:**图片模型生成** 或 **手动上传**;可替换、可重生成;**生成路径需已配置图片模型**(上传单张 ≤ 1MB、超限压缩,见 §4.1;上传不受模型门禁限) |
| FR-L5 | 库角色引入剧集 | 创建/编辑剧集角色时,可**从角色库引入**一个库角色作为该剧的剧本角色(FR-A4);引入后为该剧独立副本,可在此剧内再编辑而不影响库原件 |
| FR-L6 | 列表与检索 | 角色库列表(缩略图 + 姓名 + 是否主角);支持按姓名检索、按主角/配角筛选 |

## 3. 角色模型(库档案)

- **图片** image:单张(生成或上传);作为形象参考。
- **姓名** name:必填。
- **年龄** age:**枚举(单选)**;成员:`不详` · `幼年` · `少年` · `青年` · `中年` · `老年`(成员集可调,但限定枚举、不接受自由文本)。
- **描述** description:人设 / 性格 / 背景等自由文本。
- **是否主角** is_protagonist:布尔(主角 / 配角)。

> 库档案字段**精简于**剧集角色([analysis.md](./analysis.md) §3:含身份/动机/特质/关系/关键场次等);引入剧集后可由 FR-A5 拆解或用户补全这些富字段。

## 4. 关键能力细化

### 4.1 图片(生成 / 上传)(FR-L4)

- **生成**:调用用户当前生效的图片模型,依据「姓名 + 年龄 + 描述 + 是否主角」生成形象图;**需已配置图片模型**,否则该入口禁用并引导补配([ai-config.md](./ai-config.md) FR-C1);**生成前给成本预估与确认**(BYOK)。
- **上传**:用户手动上传图片;**单张 ≤ 1MB,超限自动压缩**至 ≤ 1MB(落盘体积上限 1MB);无需配置图片模型。支持格式(jpg / png / webp 等,具体见实现)。
- 生成与上传二选一,均**可替换 / 重生成**。

### 4.2 库 ↔ 剧集 双向流转

- **剧集 → 库(FR-L3)**:在剧集角色页对单个角色点「加入角色库」→ 按库字段抽取(姓名 / 年龄 / 描述 / 是否主角;图片若存在一并带入)→ 复制为库角色。
- **库 → 剧集(FR-L5)**:在剧集角色页点「从角色库引入」→ 选一个库角色 → 复制为该剧的剧本角色。
- 两向均为**单独复制**:不建立引用、**不做来源追溯、不联动更新**,库与剧集互不影响(见 §8)。

## 5. 业务规则

- **隔离**:库角色严格归属当前用户(NFR-7);本期不跨用户可见 / 分享(见 §8)。
- **流转语义**:库 ↔ 剧集为**单独复制**——在剧内编辑引入来的角色不回写库原件,库原件改动也不传播到已引入的剧内副本;不建立引用、不做来源追溯。
- **删除**:删除库角色**不级联**影响已引入到各剧的副本;提示用户确认。
- **图片生成门禁**:仅「生成」路径受图片模型配置门禁约束;上传不受限(同 [analysis.md](./analysis.md) FR-A7);上传单张 ≤ 1MB、超限自动压缩。
- **不去重**:加入库 / 引入剧集时**不做去重、不提示、不合并**,允许同名重复(见 §8)。

## 6. 页面与旅程

- **角色库页**:库角色列表(缩略图 + 姓名 + 主 / 配角标记);新建角色(生成 / 上传图片);编辑 / 删除;按姓名检索、主配角筛选。
- **剧集角色页(联动)**:在 [analysis.md](./analysis.md) 剧集角色管理(FR-A4)处,增加「**从角色库引入**」与「**加入角色库**」两个入口。
- **旅程(建库)**:进入角色库 → 新建角色(填姓名 / 年龄 / 描述 / 是否主角 + 生成或上传图片)→ 保存。
- **旅程(沉淀)**:在某剧集角色页 → 选中角色 →「加入角色库」→ 库中可见。
- **旅程(复用)**:新建剧集角色 →「从角色库引入」→ 选库角色 → 进入该剧。

## 7. 接口(草案;字段契约随变更定)

> 均要求携带 token;均归属当前用户。

| 资源 | 端点示例 | 说明 |
|------|----------|------|
| 角色库 | `GET/POST/PUT/DELETE /api/me/characters` · `GET /api/me/characters/:id` | 库角色 CRUD + 列表(检索 / 筛选) |
| 角色图片 | `POST /api/me/characters/:id/image` · `POST /api/me/characters/:id/image/upload` | 图片模型生成 · 手动上传(二选一,可替换;上传 ≤ 1MB、超限压缩) |
| 从剧集加入库 | `POST /api/episodes/:id/characters/:cid/promote-to-library` | 把该剧集角色加入角色库(FR-L3) |
| 从库引入剧集 | `POST /api/episodes/:id/characters`(body: `{ fromLibraryId }`)· 或 `POST /api/me/characters/:id/clone-to-episode/:epId` | 把库角色引入为该剧剧本角色(FR-L5) |

> 端点形态随变更定;`/api/me/characters` 表示「当前用户的角色库」,区别于 `/api/episodes/:id/characters`(剧集角色,见 [analysis.md](./analysis.md) §8)。

## 8. 决策记录

> 本节原 6 项待澄清已于 2026-06-26 全部裁定,落点如下;无遗留待定。

1. **「公共」范围** → **同一用户跨自身剧目的公共池,本期不做跨用户分享**(受 NFR-7 约束);用户间分享 / 市场后续再议。
2. **库 ↔ 剧集流转语义** → **单独复制**:互不影响,**不建立引用、不做来源追溯、不联动更新**。
3. **去重 / 合并** → **不做去重**:加入库 / 引入剧集时不提示、不合并,允许同名重复。
4. **图片上传约束** → 单张 **≤ 1MB**,**超限自动压缩**至 ≤ 1MB(落盘体积上限 1MB;压缩具体策略属实现)。
5. **年龄字段形态** → **枚举(单选)**,成员:`不详` · `幼年` · `少年` · `青年` · `中年` · `老年`(成员集可调,但限定枚举、不接受自由文本)。
6. **库容量上限** → **本期不设硬性上限**。
