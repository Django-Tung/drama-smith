# 用户与认证(FR-U)

> 所属:drama-smith 需求 · 功能点细化 · v0.5 · 2026-06-26
> 父文档:[system-requirements.md](../system-requirements.md)
> 关联编号:FR-U · 切面:用户体系

## 1. 概述

多用户**账号体系**:用户名 + 密码注册登录,**无需邀请码、无需邮箱验证**(本期不开邮箱)。登录成功签发**短时访问令牌** + **可吊销的刷新令牌**;后续请求以 `Authorization: Bearer <access_token>` 携带访问令牌鉴权。访问令牌过期后凭刷新令牌换新;登出吊销刷新令牌、客户端丢弃访问令牌。所有数据按账号严格隔离。令牌/哈希等技术选型见 [`architecture`](../../architecture/system-architecture.md) §4.6。

## 2. 需求条目

| 编号 | 需求 | 说明 |
|------|------|------|
| FR-U1 | 注册与登录 | 用户名 + 密码注册,**无需邀请码、无需邮箱验证**;用户名唯一(建议 3–32 字符、字母数字下划线)。密码须达**中等强度**(长度 8–64、须含字母 + 数字,不强制大小写/特殊字符)。登录成功签发访问令牌 + 刷新令牌 |
| FR-U2 | 用户级数据隔离 | 剧本、分析结果、AI 配置严格按账号隔离,互不可见 |
| FR-U3 | 个人数据管理 | 用户可查看/删除自己的剧本与分析结果(满足可清理) |
| FR-U4 | 登出与会话 | 支持登出(吊销刷新令牌、客户端丢弃访问令牌);访问令牌短时过期,凭刷新令牌刷新(机制见 §5) |

## 3. 页面

- **注册 / 登录页**:账号注册、登录(签发 token)。
- 个人信息维护归入设置页,见 [`ai-config.md`](./ai-config.md)。
- 个人数据(剧本/分析记录)的查看与删除(FR-U3)入口见 [`analysis.md`](./analysis.md)「我的历史」。

## 4. 接口(草案;字段契约随变更定)

| 资源 | 端点示例 | 说明 |
|------|----------|------|
| 认证 | `POST /api/auth/register` · `/login` · `/logout` · `/refresh` | 注册、登录(签发访问令牌 + 刷新令牌)、登出(吊销刷新令牌)、刷新(凭刷新令牌换新访问令牌) |
| 当前用户 | `GET /api/me` | 取当前用户与配置完成度(**是否已配置文本模型**;图片/视频可选,未配则相应功能门禁,见 [ai-config.md](./ai-config.md) FR-C1) |

> 所有非认证端点默认要求以 `Authorization: Bearer <access_token>` 携带访问令牌(对应 NFR-6)。

## 5. 安全要求

> 令牌格式、哈希算法、客户端存储等技术选型见 [`architecture`](../../architecture/system-architecture.md) §4.6;本节列**安全需求与策略**。

- **密码**:加盐哈希(单向、不可逆)存储;**中等强度**——长度 8–64、须同时含字母与数字(不强制大小写/特殊字符);本期无邮箱/邀请,登录失败不引入验证码。
- **访问令牌**:自包含、**短时**有效、无服务端状态;签名密钥来自环境变量;令牌不得明文落盘/入日志(NFR-6)。
- **刷新令牌**:服务端**可吊销**、不存明文;登出即吊销。
- **登录防爆破**:**失败次数锁定,仅按账号维度**(默认阈值见 architecture §4.6);**不按 IP 限流、不用指数退避**,避免 NAT / 共享 IP 误伤正常用户。
- **客户端令牌**:访问/刷新令牌**分层存储**,降低被持久窃取的风险;相应收敛 XSS 面。
- **多租户隔离**:所有数据访问强制带用户归属,杜绝越权读取他人数据(NFR-7)。

## 6. 决策记录

**已澄清(2026-06-25)**

1. **访问令牌 = JWT(HS256)**;另配**不透明刷新令牌**承担续期与可吊销登出,见 §5。
2. **注册无需邀请码、无需邮箱验证**(本期不开邮箱);**密码中等强度**(≥ 8、含字母 + 数字),见 §5。
3. **登录防爆破**:启用失败次数锁定、**仅按账号维度**;不按 IP、不用指数退避(见 §5)。
4. **客户端令牌存储分层**:访问令牌 → `localStorage`;刷新令牌 → 内存 / `sessionStorage`;不采用 `httpOnly` Cookie(见 §5)。
